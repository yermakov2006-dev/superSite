001 / Введение

Кто это? Определение и границы ответственности

Frontend-разработчик — специалист, который реализует пользовательский интерфейс веб-приложения. Он берёт дизайн-макет и превращает его в код, работающий в браузере. В зону ответственности входит: вёрстка, логика интерфейса, работа с данными от сервера, оптимизация загрузки, обеспечение доступности (a11y) и кросс-браузерной совместимости.

В отличие от верстальщика (который только раскладывает блоки по сетке), фронтенд-разработчик пишет полноценную клиентскую логику: управляет состоянием приложения, обрабатывает пользовательские сценарии, работает с API, настраивает сборку и деплой.

003 / Обязанности

Что реально делает? Не «пишет код», а решает проблемы

01

Строит интерфейсы

Собирает приложение из компонентов. Управляет состоянием (Redux, Zustand, Pinia или signals). Настраивает роутинг. Следит, чтобы интерфейс не перерисовывался без необходимости — мемоизация, виртуализация списков, оптимизация ре-рендеров.

02

Работает с данными

Ходит в API (REST / GraphQL), кеширует ответы (React Query / SWR / Apollo), обновляет данные через WebSocket или SSE. Обрабатывает загрузку, ошибки, ретраи. Делает optimistic updates для отзывчивости.

03

Оптимизирует

Следит за Core Web Vitals: LCP, CLS, INP. Дробит бандл через code splitting, грузит лениво (lazy loading), сжимает изображения (WebP/AVIF), настраивает кеширование через Service Worker.

04

Обеспечивает качество

Пишет тесты: модульные (Vitest), компонентные (Testing Library), E2E (Playwright). Внедряет линтеры и форматтеры. Проводит code review. Проверяет доступность: семантика, aria, клавиатура, WCAG 2.1 AA.

005 / Карьера

Путь в профессии От джуна до лида — что меняется

  • JuniorВёрстка по Figma, простые компоненты, основы JS и один фреймворк. Нужен код-ревью. 0–1.5 года. 60–120k.
  • MiddleУверенный фреймворк, сборка, тесты, ревью. Понимает SSR vs SPA vs SSG. Рефакторит без страха. 1.5–5 лет. 150–300k.
  • SeniorАрхитектура фронтенда, выбор стека, performance. Участвует в планировании. Знает бэкенд на уровне API. 5+ лет. 300–600k+.
  • LeadУправляет командой, нанимает, онбордит, выстраивает процессы. Код редко — в основном ревью и прототипы.

006 / Тренды

Что прямо сейчас Куда движется индустрия

  • Server ComponentsReact Server Components (RSC) меняют подход: часть компонентов рендерится на сервере без отправки JS на клиент. Меньше бандла, быстрее загрузка.
  • AI в интерфейсеЧат-интерфейсы, AI-автодополнение, генерация UI по промпту. Stream через SSE — стандартный паттерн.
  • WebGPUAPI для вычислений на GPU в браузере. Тяжёлая 3D-графика, симуляции, ML-инференс на клиенте без плагинов.
  • Меньше JSIsland architecture (Astro), partial hydration (Qwik), компилируемые фреймворки (Svelte, Solid). Меньше JS = быстрее сайт.

Источники: State of JS 2024, HTTP Archive, Web Almanac, JetBrains.

007 / Мифы и FAQ

Чего не надо говорить Популярные заблуждения и ответы

«Фронтенд — это же просто вёрстка»+
Фронтенд включает управление состоянием, оптимизацию производительности, тестирование, CI/CD. Вёрстка — 20–30% работы. Остальное — инженерия.
«Можно без математики»+
Анимации, алгоритмы поиска/фильтрации, сложность O(n), работа с canvas и WebGL — математика нужна регулярно. Хотя бы на уровне алгоритмов.
«Все фреймворки — одно и то же»+
React — библиотека без мнения. Vue — фреймворк с магией. Svelte — компилятор. Solid — без виртуального DOM. Разный подход к реактивности.
«Достаточно знать React»+
React — только инструмент. Без понимания DOM, HTTP, accessibility, рендеринга, бандлинга — вы просто пользователь чужого API.
«AI скоро заменит фронтендеров»+
AI ускоряет написание кода, но не проектирование архитектуры, не принятие компромиссов, не понимание пользовательского опыта. Чем сложнее система — тем нужнее человек.
