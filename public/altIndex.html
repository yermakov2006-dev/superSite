Duties

Что реально делает фронтендер?

Не «пишет код», а решает проблемы бизнеса и пользователей.

Build. Ship. Repeat.

Собирает интерфейсы из компонентов. Работает с данными через API. Оптимизирует загрузку. Обеспечивает качество тестами и код-ревью.

Фронтенд — это не про «натянуть дизайн». Это про архитектуру, состояние, производительность и пользовательский опыт.

Career Path

От джуна до лида

Что меняется на каждом этапе и сколько это стоит.

  • JuniorВёрстка по Figma, простые компоненты, JS + один фреймворк. Нужен код-ревью. 0–1.5 года. 60–120k.
  • MiddleУверенный фреймворк, сборка, тесты, ревью. Понимает SSR vs SPA vs SSG. 1.5–5 лет. 150–300k.
  • SeniorАрхитектура, стек, performance. Участвует в планировании. Знает бэкенд. 5+ лет. 300–600k+.
  • LeadУправляет командой, нанимает, процессы. Код — в основном ревью и прототипы.

Trends

Куда движется индустрия

Ключевые направления, которые определяют будущее фронтенда.

  • Server ComponentsReact Server Components: часть компонентов рендерится на сервере. Меньше JS на клиенте, быстрее загрузка.
  • AI в интерфейсеЧат-интерфейсы, AI-автодополнение, генерация UI по промпту. Stream через SSE.
  • WebGPUAPI для вычислений на GPU в браузере. 3D-графика, симуляции, ML-инференс.
  • Меньше JSIsland architecture (Astro), partial hydration (Qwik), компилируемые фреймворки.

Источники: State of JS 2024, HTTP Archive, Web Almanac, JetBrains.

FAQ

Чего не надо говорить

Популярные заблуждения о профессии и развёрнутые ответы.

«Фронтенд — это же просто вёрстка»
Фронтенд включает управление состоянием, оптимизацию производительности, тестирование, CI/CD. Вёрстка — 20–30% работы.
«Можно без математики»
Анимации, алгоритмы поиска/фильтрации, O(n), canvas, WebGL — математика нужна регулярно.
«Все фреймворки — одно и то же»
React — библиотека без мнения. Vue — фреймворк с магией. Svelte — компилятор. Solid — без виртуального DOM.
«Достаточно знать React»
React — инструмент. Без DOM, HTTP, accessibility, бандлинга — вы просто пользователь API.
«AI скоро заменит фронтендеров»
AI ускоряет код, но не архитектуру, не компромиссы, не пользовательский опыт.
