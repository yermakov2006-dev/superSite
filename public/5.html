001 / Введение

Кто это?Определение и границы ответственности

Frontend-разработчик — специалист, который реализует пользовательский интерфейс веб-приложения. Он берёт дизайн-макет и превращает его в код, работающий в браузере. В зону ответственности входит: вёрстка, логика интерфейса, работа с данными от сервера, оптимизация загрузки, обеспечение доступности (a11y) и кросс-браузерной совместимости.

В отличие от верстальщика (который только раскладывает блоки по сетке), фронтенд-разработчик пишет полноценную клиентскую логику: управляет состоянием приложения, обрабатывает пользовательские сценарии, работает с API, настраивает сборку и деплой.

003 / Обязанности

Что реально делает?Не «пишет код», а решает проблемы

01

Строит интерфейсы

Собирает приложение из компонентов. Управляет состоянием (Redux, Zustand, Pinia или signals). Настраивает роутинг. Мемоизация, виртуализация списков, оптимизация ре-рендеров.

02

Работает с данными

Ходит в API (REST / GraphQL), кеширует ответы (React Query / SWR / Apollo), обновляет через WebSocket или SSE. Обрабатывает ошибки, ретраи, optimistic updates.

03

Оптимизирует

Следит за Core Web Vitals: LCP, CLS, INP. Дробит бандл через code splitting, ленивая загрузка, WebP/AVIF, Service Worker кеширование.

04

Обеспечивает качество

Тесты: Vitest, Testing Library, Playwright. Линтеры, code review, доступность: семантика, aria, клавиатура, WCAG 2.1 AA.

005 / Карьера

Путь в профессииОт джуна до лида — что меняется

  • JuniorВёрстка по Figma, простые компоненты, основы JS и один фреймворк. Нужен код-ревью. 0–1.5 года. 60–120k.
  • MiddleУверенный фреймворк, сборка, тесты, ревью. Понимает SSR vs SPA vs SSG. 1.5–5 лет. 150–300k.
  • SeniorАрхитектура, выбор стека, performance. Участвует в планировании. Знает бэкенд. 5+ лет. 300–600k+.
  • LeadУправляет командой, нанимает, онбордит, процессы. Код редко — в основном ревью и прототипы.

006 / Тренды

Что прямо сейчасКуда движется индустрия

  • Server ComponentsReact Server Components (RSC): часть компонентов рендерится на сервере без отправки JS на клиент. Меньше бандла, быстрее загрузка.
  • AI в интерфейсеЧат-интерфейсы, AI-автодополнение, генерация UI по промпту. Stream через SSE — стандартный паттерн.
  • WebGPUAPI для вычислений на GPU в браузере. 3D-графика, симуляции, ML-инференс на клиенте.
  • Меньше JSIsland architecture (Astro), partial hydration (Qwik), компилируемые фреймворки (Svelte, Solid).

Источники: State of JS 2024, HTTP Archive, Web Almanac, JetBrains.

007 / Мифы и FAQ

Чего не надо говоритьПопулярные заблуждения и ответы

«Фронтенд — это же просто вёрстка»
Фронтенд включает управление состоянием, оптимизацию производительности, тестирование, CI/CD. Вёрстка — 20–30% работы.
«Можно без математики»
Анимации, алгоритмы поиска/фильтрации, O(n), canvas, WebGL — математика нужна регулярно.
«Все фреймворки — одно и то же»
React — библиотека без мнения. Vue — фреймворк с магией. Svelte — компилятор. Solid — без виртуального DOM.
«Достаточно знать React»
React — инструмент. Без DOM, HTTP, accessibility, бандлинга — вы просто пользователь API.
«AI скоро заменит фронтендеров»
AI ускоряет код, но не архитектуру, не компромиссы, не пользовательский опыт. Чем сложнее — тем нужнее человек.
